# HIMALAYA — Locked Product Doctrine

## READ BEFORE ANY WORK
Start every session by reading: `docs/himalaya/START_HERE.md`

## What Is Himalaya
Himalaya is a **guided business operating system**. Not a stack of tools. It helps people start, structure, optimize, and scale businesses through guided AI systems.

## Sacred Sequence
```
Entry → Diagnosis → Strategy → Generation → Optimization
```
Generation never fires without diagnosis and strategy first.

## Decision Filter
> Does this reduce confusion, create structure, or give the user a faster path forward?

If NO → defer. If UNCLEAR → ask first.

## Build Rules
1. Map every task to the roadmap
2. Follow the sacred sequence
3. Integrate existing code — don't rebuild what works
4. Match the UX doctrine
5. Do not add disconnected features
