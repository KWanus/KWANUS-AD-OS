# FUNDING_GUIDANCE_SPEC.md

## Purpose
Guide users on when and how to pursue funding.

---

## Core Truth
Most users fail because they apply too early or incorrectly.

---

## Readiness Levels

### Not Ready
- major negatives present
- poor utilization
- weak structure

### Improving
- cleanup in progress
- partial strength

### Near Ready
- strong structure
- few blockers

### Ready
- optimized profile
- high likelihood of approval

---

## Decision Logic

System evaluates:
- score range (not sole factor)
- profile structure
- utilization
- account mix
- recent activity
- inquiries

---

## Output

User sees:
- readiness level
- why they are at that level
- what is blocking them
- what to fix first
- when to apply

---

## Application Guidance

System may suggest:
- spacing between applications
- type of lenders
- timing windows

---

## Guardrail
Never:
- guarantee approval
- suggest reckless applications
- push users before readiness
