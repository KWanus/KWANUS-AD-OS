# HIMALAYA — Phase 1 Specification

## Promise

> "Choose or bring a business idea, and Himalaya gives you a high-converting foundation."

## Two Paths

### Path A: Start from Scratch
User has no business. Himalaya helps them choose and build.

### Path B: Improve Existing Business
User has a business. Himalaya scans, diagnoses, and rebuilds what's weak.

---

## Path A: Start from Scratch

### Input Flow
1. User selects "Starting from scratch"
2. Choose business model (service, e-commerce, coaching, agency, freelance, affiliate, info product)
3. Choose niche (guided selection or free input)
4. Define goal (first client, first $1K, replace income, scale to $10K/mo)
5. Optional: describe dream business in plain language

### Diagnosis
- Business model viability assessment
- Niche clarity score
- Goal-to-model alignment check
- Resource requirements estimate

### Strategy Output
- Recommended business structure
- What to build first (prioritized)
- What not to build yet
- Expected timeline to first result

### Generation Output
| Asset | Description |
|-------|-------------|
| Business positioning summary | Who you are, who you serve, how you're different |
| Niche clarity doc | Market, pain points, desires, buying triggers |
| Ideal customer profile | Demographics, psychographics, behavior patterns |
| Offer concept | What you sell, how it's packaged, why it's compelling |
| Website blueprint | Page structure, section order, content direction |
| Homepage copy | Full homepage: headline, sections, CTA, trust elements |
| Brand direction | Voice, tone, visual direction, positioning statement |
| First email/SMS sequence | 3-5 message welcome + nurture starter |
| Marketing angle direction | Top 3 angles for ads/content/outreach |
| Action roadmap | Step-by-step: what to do this week, this month, this quarter |

---

## Path B: Improve Existing Business

### Input Flow
1. User selects "I have a business"
2. Enter website URL and/or describe business
3. Optional: upload additional assets (ad copy, email, offer page)
4. Answer quick diagnostic questions (niche, revenue, biggest challenge)

### Diagnosis
Run full Business Diagnosis Engine:
- Website scan (structure, copy, trust, CTA, conversion flow)
- Offer analysis (clarity, value, objection handling)
- Trust assessment (social proof, credibility indicators)
- Conversion flow audit (from landing to action)
- Follow-up assessment (email, automation, retention)
- Brand clarity check (positioning, differentiation)

### Diagnosis Output
| Output | Description |
|--------|-------------|
| Business audit report | Overall health score + breakdown |
| Weak points | Ranked by impact on growth |
| Trust/conversion breakdown | What's helping, what's hurting |
| Missed opportunities | What's not being done that should be |
| Fastest wins | Highest ROI fixes, ordered |

### Strategy Output
- What to rebuild first (prioritized action plan)
- What to keep (what's working)
- What to defer (not worth fixing now)
- Expected impact per change

### Generation Output
| Asset | Description |
|-------|-------------|
| New website blueprint | Restructured for conversion |
| Rewritten homepage | Copy rebuilt around diagnosis findings |
| Ad angle suggestions | Top angles based on audience and offer |
| Email follow-up suggestions | Sequence improvements or new sequences |
| Optimization roadmap | Ordered list of changes with impact estimates |

---

## Phase 1 Validation Criteria

Phase 1 is complete when:
- [ ] A new user can enter the "start from scratch" flow and receive all Phase 1 generation outputs
- [ ] An existing business user can enter the "improve" flow and receive a full diagnosis + generation outputs
- [ ] The strategy layer visibly prioritizes and sequences recommendations
- [ ] Every generated asset includes context-specific reasoning
- [ ] The experience feels guided, not overwhelming
- [ ] Both paths connect to the dashboard with clear next actions
- [ ] Beginner mode and operator mode feel different
- [ ] No feature exists that doesn't map to Entry → Diagnosis → Strategy → Generation

## What Phase 1 Does NOT Include
- Optimization Engine (Phase 2)
- Business model specialization (Phase 3+)
- Advanced automation builder (Phase 3+)
- CRM pipeline management (Phase 2)
- Ad platform integrations (Phase 3+)
- Payment/billing system (Phase 2)
