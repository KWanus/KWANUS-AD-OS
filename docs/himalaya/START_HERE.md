# HIMALAYA — START HERE

> This is the canonical control file. Read this before touching any code.

## What Is Himalaya?

Himalaya is a **guided business operating system**. Not a stack of tools. Not a dashboard with buttons. Not a generic AI wrapper.

It helps people start, structure, optimize, and scale businesses through guided AI systems that reduce friction and improve execution.

## Core Promise

> "I came in confused, and now I have clarity, structure, and assets."

## Who It Serves

| User Type | What They Need |
|-----------|---------------|
| No business, no product, no structure | Choose a path. Build a foundation. |
| Business exists but bad systems | Diagnose. Fix what's killing growth. |
| Business exists but weak marketing | Scan. Rebuild the parts that matter. |
| Needs help choosing the right path | Guide first, then build. |

## Core Sequence (Sacred)

```
Entry → Diagnosis → Strategy → Generation → Optimization
```

This sequence is law. Generation never happens before diagnosis and strategy.

## 5 Core Engines

1. **Entry Engine** — How users enter the system with minimal friction
2. **Business Diagnosis Engine** — Understand the user's situation deeply
3. **Strategy Engine** — Decide what to build, in what order, and why
4. **Generation Engine** — Create the actual assets and systems
5. **Optimization Engine** — Improve what's built over time
