# SYSTEM_STATE.md

## Current Phase
Phase 1 — Core Outcome Engine (Upload -> Analyze -> Decide -> Generate -> Guide)

## Current Objective
Stabilize the full end-to-end user loop from report upload to structured next actions and document generation.

## Canonical Product Definition
KWANUS is a financial outcome engine that creates order before money moves.
It helps users understand what matters, what to do next, and when to move.

## Active Core Loop
1. Upload report
2. Parse and extract data
3. Identify key issues
4. Prioritize sequence
5. Generate guidance and documents
6. Present next steps
7. Track progress
8. Evaluate readiness

## Live Engines (Phase 1)
- Intake Engine
- Parsing Engine
- Strategy Engine
- Document Engine
- Explanation Engine
- Readiness Engine (basic version)

## Locked Decisions
- Consultant-only posture (no execution on behalf of user)
- No guarantees, no legal claims
- Calm, premium, minimal UI
- Focus on sequence over features
- Repair + Build + Readiness (not just disputes)

## Known Weak Points

- Readiness scoring not deeply calibrated
- Explanation layer may be too shallow
- No strong "what to add" system yet
- Limited outcome memory
