# Build Doctrine

## Rules

1. DO NOT overbuild
2. DO NOT redesign existing system
3. ADD modular layers only
4. FRONTEND = simple
5. BACKEND = intelligent

---

## UX Standard
- 2–4 clicks max
- one action per screen
- no clutter

---

## Development Standard
- phase-based only
- each phase must work independently
- no dependencies on unfinished features
