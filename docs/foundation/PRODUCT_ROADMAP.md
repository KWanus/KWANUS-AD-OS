# KWANUS — Product Roadmap

## Phase 1: Foundation & Repair Engine
**Status: Active**

- User onboarding (profile intake, credit report upload)
- Credit report parsing and item extraction
- Dispute identification engine (flag inaccuracies, outdated items, duplicates)
- Dispute letter generation (educational, compliance-safe)
- Dashboard: user status, active disputes, profile snapshot
- Database schema: users, credit items, disputes, timeline events

## Phase 2: Profile Build Engine
**Status: Queued**

- Credit profile strength scoring
- Account mix analysis
- Utilization tracking and guidance
- Tradeline education modules
- Authorized user strategy guidance
- Age-of-accounts visibility

## Phase 3: Optimization Engine
**Status: Queued**

- Score factor breakdown and prioritization
- Timing recommendations (when to apply, when to wait)
- Hard inquiry tracking and strategy
- Balance optimization suggestions
- Profile presentation scoring

## Phase 4: Funding Readiness Engine
**Status: Queued**

- Readiness assessment (are you ready to apply?)
- Documentation checklist
- Lender-profile matching (educational, not brokering)
- Funding timing guidance
- Application sequencing recommendations

## Phase 5: Funding Guidance
**Status: Queued**

- Funding opportunity awareness (educational)
- Application strategy guidance
- Approval likelihood indicators
- Post-funding profile protection guidance
- Ongoing monitoring recommendations

## Phase 6: Intelligence & Memory
**Status: Queued**

- Pattern recognition across user actions
- Outcome tracking (what worked, what didn't)
- Personalized action prioritization
- Retention loops and re-engagement
- Progress milestones and celebration moments

## Phase 7: Scale & Polish
**Status: Queued**

- Performance optimization
- UI/UX refinement
- Accessibility compliance
- Mobile optimization
- Analytics and conversion tracking

---

## Roadmap Rules

1. Phases execute in order. No skipping.
2. Each phase must be validated before the next begins.
3. Features not on this roadmap require explicit approval and must map to the core loop.
4. The core loop is: **Repair -> Build -> Optimize -> Prepare -> Fund**
5. Every phase must strengthen this loop, not add disconnected capabilities.
